## 二、Skills 升级路线图

### Skill 1：JD Analyzer → JD Intelligence Engine

```
现状：
  输入 JD → 输出结构化字段

升级后：
  输入 JD → 结构化字段 + 超预期公司画像 + JD 吸引力评分

新增能力：
├── 「JD 隐含偏好挖掘」
│   例："有创业心态" = 能接受低薪高股
├── 「公司超预期亮点注入」
│   从公开信息给 JD 自动标注公司亮点
├── 「候选人稀缺性评估」
│   这个 JD 要求的人才，在市场上大约有多少人？
├── 「薪资竞争力校准」
│   JD 写的薪资 vs 市场上实际薪资，是否有水分？
└── 「匹配难度评级」
    容易 / 中等 / 困难（影响 HR 的期望管理）
```

### Skill 2：Resume Parser → Candidate Intelligence Engine

```
现状：
  输入简历 → 输出候选人画像

升级后：
  输入简历 → 画像 + 全网背景补充 + 超预期亮点发现

新增能力：
├── 「全网背景补充」
│   LinkedIn + GitHub + 脉脉 + 论文专利 自动交叉验证
├── 「薪资/级别预测」
│   没有写期望薪资？根据履历自动估算
├── 「风险预警」
│   频繁跳槽 / 职业断层 / 学历存疑 → 主动标注
├── 「超预期亮点发现」
│   开源 star 数 / 顶会论文 / 团队规模等隐藏信息
├── 「求职意向推测」
│   候选人没明说？根据轨迹推测可能的跳槽方向
└── 「文化契合度」
    候选人风格 vs 公司文化的隐性匹配度
```

### Skill 3：Group Info Extractor → Group Intelligence Hub

```
现状：
  从群聊提取 JD + 候选人

升级后：
  群聊信息收集 + 智能分类 + 匹配预热 + 双向推荐

新增能力：
├── 「附件解析」
│   图片 JD、PDF 简历、语音消息 → 自动识别并解析
├── 「需求紧迫度识别」
│   急招 / 高端岗 / HC 明确 → 主动加权
├── 「候选人去重 + 合并」
│   同一个人在多个群出现 → 合并画像
├── 「顾问关系图谱」
│   这条 JD 是谁发的？顾问的风格是什么？
├── 「超预期线索捕获」
│   群聊中的「顺便提一句」往往是最好的洞察
└── 「预匹配推送」
    提取后自动对 JD/候选人做一轮初步匹配
```

### Skill 4：CGL Email Template → Smart Outreach Engine

```
现状：
  输入信息 → 基于公式生成 InMail

升级后：
  超预期亮点自动注入 + 双向定制 + 追踪机制

新增能力：
├── 「超预期亮点自动注入」
│   Discovery Radar 发现的亮点 → 自动写入邮件
├── 「候选人定制化」
│   根据候选人的年龄/职级/职能，自动调整语气和卖点
├── 「邮件变体生成」
│   同一封邮件生成 3 个变体（简洁版 / 完整版 / 高管版）
├── 「邮件效果追踪」
│   发送后自动追踪是否被打开 / 被回复
├── 「跟进时机建议」
│   什么时候跟？根据候选人的 LinkedIn 活跃时间判断
└── 「拒绝话术库」
    候选人说不合适？系统自动生成留人口碑的话术
```

### Skill 5：新建 Discovery Radar（填补空缺）

```
定位：双向背景尽调 Skill，共用于招聘者和求职者两侧

招聘者侧 → 查候选人：
├── LinkedIn：职位轨迹、推荐语、人脉网络
├── GitHub：项目活跃度、技术栈贡献、被引用情况
├── 脉脉：行业口碑、同事评价
├── 论文/专利：学术深度、技术判断力
└── 新闻舆情：媒体曝光、演讲分享

求职者侧 → 查公司：
├── 融资信息：轮次、估值、投资方
├── 团队背景：CTO/高管 LinkedIn、技术团队规模和质量
├── 财务健康：财报摘要、员工规模变化
├── 舆情分析：脉脉/看准员工评价、离职率、加班文化
└── 竞品对比：行业地位、市场份额、差异化

输出格式：
├── 超预期亮点清单（最多 5 条，每条带来源）
├── 风险提示（如有）
└── 匹配置信度：High / Medium / Low
```

## 四、Skills 升级优先级决策矩阵

| 优先级 | Skill 升级方向 | 理由 | 依赖关系 |
|--------|---------------|------|---------|
| **P0** | Discovery Radar（新建） | 双向撮合的核心，其他 Skills 都依赖它 | 无 |
| **P0** | JD Analyzer → JD Intelligence | 超预期亮点是撮合公式的关键输入 | 依赖 Discovery Radar |
| **P0** | Resume Parser → Candidate Intelligence | 同上 | 依赖 Discovery Radar |
| **P1** | Group Info Extractor → Group Hub | 提高群聊信息收集效率 | 独立 |
| **P1** | Email Template → Smart Outreach | 邮件带超预期亮点，直接提升回复率 | 依赖 Discovery Radar |

> **建议路径**：先做 P0 三个，形成闭环再迭代 P1

## 五、数据来源决策

| 选项 | 方案 | 优势 | 劣势 | 建议 |
|------|------|------|------|------|
| **A** | 纯 API 调用 | 稳定、可控 | LinkedIn/脉脉需要合作 | 技术岗用 GitHub API |
| **B** | 浏览器自动化 | 通用性强 | 账号封禁风险 | 高价值信息人工补充 |
| **C** | 混合模式 ✓ | 平衡风险和覆盖 | 需要维护多套 | **推荐** |

**推荐：选项C（混合模式）**
- 技术岗 → GitHub API（免费，P0 优先覆盖）
- 通用搜索 → Tavily API（已有 Key）
- 高价值信息 → 人工补充

## 六、个性化 Level 决策

| Level | 能力 | 效果 | 成本 | 建议 |
|-------|------|------|------|------|
| **Level 1** | 超预期亮点注入 | 回复率 +20-30% | 低 | ✅ **快速见效** |
