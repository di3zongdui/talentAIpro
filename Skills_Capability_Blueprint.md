### 🧪 JD Intelligence Engine v2.0 × Levi Li 实测

**测试数据：**
- 候选人：Levi Li（985本硕，12年硬件一号位，P9对标，具身智能创业）
- JD：具身智能HR负责人，北京，30-50万

**实测结果：**

| 能力 | 结果 | 验证 |
|------|------|------|
| JD隐含偏好挖掘 | ✅ 有效 | 识别出"技术理解力是隐性必备项" |
| 薪资竞争力校准 | ✅ 有效 | 发现JD低于市场20-30% |
| 匹配难度评级 | ✅ 有效 | 困难★★★★☆，原因清晰 |
| 超预期公司亮点 | ⚠️ 待优化 | Discovery Radar需换查询词 |
| 候选人稀缺性评估 | ✅ 有效 | 估算池约20-50人 |

**撮合分析：**
- 招聘者满意分：40/100（薪资严重不匹配）
- 候选人满意分：85/100（背景超出JD要求）
- 综合撮合分：37/100（未触发阈值75）
- 结论：薪资差距过大，需猎头主动澄清

## 五、Skills 升级路线图（详细版）

### Skill 1：JD Analyzer → JD Intelligence Engine

```
现状：
  输入 JD → 输出结构化字段

升级后：
  输入 JD → 结构化字段 + 超预期公司画像 + JD 吸引力评分

新增能力：
├── 「JD 隐含偏好挖掘」
│   例："有创业心态" = 能接受低薪高股
├── 「公司超预期亮点注入」
│   从公开信息给 JD 自动标注公司亮点
├── 「候选人稀缺性评估」
│   这个 JD 要求的人才，在市场上大约有多少人？
├── 「薪资竞争力校准」
│   JD 写的薪资 vs 市场上实际薪资，是否有水分？
└── 「匹配难度评级」
    容易 / 中等 / 困难（影响 HR 的期望管理）
```

### Skill 2：Resume Parser → Candidate Intelligence Engine

```
现状：
  输入简历 → 输出候选人画像

升级后：
  输入简历 → 画像 + 全网背景补充 + 超预期亮点发现

新增能力：
├── 「全网背景补充」
│   LinkedIn + GitHub + 脉脉 + 论文专利 自动交叉验证
├── 「薪资/级别预测」
│   没有写期望薪资？根据履历自动估算
├── 「风险预警」
│   频繁跳槽 / 职业断层 / 学历存疑 → 主动标注
├── 「超预期亮点发现」
│   开源 star 数 / 顶会论文 / 团队规模等隐藏信息
├── 「求职意向推测」
│   候选人没明说？根据轨迹推测可能的跳槽方向
└── 「文化契合度」
    候选人风格 vs 公司文化的隐性匹配度
```

### Skill 3：Group Info Extractor → Group Intelligence Hub

```
现状：
  从群聊提取 JD + 候选人

升级后：
  群聊信息收集 + 智能分类 + 匹配预热 + 双向推荐

新增能力：
├── 「附件解析」
│   图片 JD、PDF 简历、语音消息 → 自动识别并解析
├── 「需求紧迫度识别」
│   急招 / 高端岗 / HC 明确 → 主动加权
├── 「候选人去重 + 合并」
│   同一个人在多个群出现 → 合并画像
├── 「顾问关系图谱」
│   这条 JD 是谁发的？顾问的风格是什么？
├── 「超预期线索捕获」
│   群聊中的「顺便提一句」往往是最好的洞察
└── 「预匹配推送」
    提取后自动对 JD/候选人做一轮初步匹配
```

### Skill 4：CGL Email Template → Smart Outreach Engine

```
现状：
  输入信息 → 基于公式生成 InMail

升级后：
  超预期亮点自动注入 + 双向定制 + 追踪机制

新增能力：
├── 「超预期亮点自动注入」
│   Discovery Radar 发现的亮点 → 自动写入邮件
├── 「候选人定制化」
│   根据候选人的年龄/职级/职能，自动调整语气和卖点
├── 「邮件变体生成」
│   同一封邮件生成 3 个变体（简洁版 / 完整版 / 高管版）
├── 「邮件效果追踪」
│   发送后自动追踪是否被打开 / 被回复
├── 「跟进时机建议」
│   什么时候跟？根据候选人的 LinkedIn 活跃时间判断
└── 「拒绝话术库」
    候选人说不合适？系统自动生成留人口碑的话术
```

### Skill 5：Discovery Radar（新建）

```
定位：双向背景尽调 Skill，共用于招聘者和求职者两侧

招聘者侧 → 查候选人：
├── LinkedIn：职位轨迹、推荐语、人脉网络
├── GitHub：项目活跃度、技术栈贡献、被引用情况
├── 脉脉：行业口碑、同事评价
├── 论文/专利：学术深度、技术判断力
└── 新闻舆情：媒体曝光、演讲分享

求职者侧 → 查公司：
├── 融资信息：轮次、估值、投资方
├── 团队背景：CTO/高管 LinkedIn、技术团队规模和质量
├── 财务健康：财报摘要、员工规模变化
├── 舆情分析：脉脉/看准员工评价、离职率、加班文化
└── 竞品对比：行业地位、市场份额、差异化

输出格式：
├── 超预期亮点清单（最多 5 条，每条带来源）
├── 风险提示（如有）
└── 匹配置信度：High / Medium / Low
```

### Skill 6：Deal Tracker（新建）

```
定位：完成闭环的最后一步

功能：
├── 候选人投递后状态跟踪
├── 撮合分计算和监控
├── 跟进提醒（时间/渠道/内容）
├── 关闭原因记录
└── 数据分析（成功率/周期/渠道效果）
```

## 六、数据来源决策（实测更新）

| 选项 | 方案 | 验证结果 |
|------|------|---------|
| **Tavily API** | 通用搜索 | ⚠️ 对中文内容覆盖不足，返回0条 |
| **web_search（miaoda）** | 通用搜索 | ✅ 对中文内容效果更好 |
| **GitHub API** | 技术岗数据 | ✅ 免费，覆盖技术候选人 |

**推荐：混合模式**
- 中文内容 → web_search（miaoda）
- 技术岗 → GitHub API
- 高价值信息 → 人工补充
