## 2. 对话数据（Message Data）

### 获取渠道

| 平台 | 对话类型 | 数据内容 |
|------|---------|---------|
| **猎聘** | 求职者和HR的聊天 | 消息内容/时间/附件/已读未读 |
| **Boss直聘** | 直聊记录 | 实时聊天/沟通过程/候选人回复率 |
| **领英** | InMail/消息 |  recruiter沟通/内推消息 |
| **脉脉** | 脉脉消息 | 职场社交沟通/问答互动 |

### 对话数据结构

```python
@dataclass
class ConversationData:
    platform: str                    # 平台来源
    conversation_id: str             # 对话ID
    participants: List[str]         # 参与者(open_id/手机号)

    # 消息流
    messages: List[Message] = field(default_factory=list)
    """
    - content: str                  # 消息内容
    - sender: str                   # 发送者
    - timestamp: datetime           # 发送时间
    - type: MessageType             # text/image/document/offer
    - is_read: bool                 # 是否已读
    """

    # 对话统计
    total_messages: int             # 总消息数
    candidate_reply_rate: float      # 候选人回复率
    avg_response_time: float         # 平均响应时间(小时)

    # 关键事件
    interview_scheduled: bool        # 是否约过面试
    offer_discussed: bool            # 是否谈过Offer
    final_status: str                # 最终状态(accepted/rejected/pending)
```

### 对话数据价值

```
┌─────────────────────────────────────────────────────────────────┐
│                   对话数据 → 候选人洞察                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  回复率 < 30%           → 🟡 候选人可能不积极                    │
│  回复率 > 80%           → 🟢 候选人活跃、意向强                 │
│                                                                 │
│  平均响应 > 24小时       → 🟡 候选人可能在犹豫                  │
│  平均响应 < 2小时        → 🟢 候选人急迫、意向强                │
│                                                                 │
│  谈过Offer但未入职       → 🔴 风险候选人（可能泡汤）             │
│  多次约面未成            → 🟡 时间难约、需要备选                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 3. 行为数据（Behavioral Data）

### 获取渠道

| 平台 | 行为类型 | 数据内容 |
|------|---------|---------|
| **猎聘** | 求职行为 | 浏览记录/搜索关键词/申请职位/投递时间/简历更新 |
| **Boss直聘** | 活跃数据 | 登录频率/在线时长/主动沟通/求职意向标签 |
| **领英** | 职业动态 | profile浏览量/帖子互动/人脉动态/职位搜索 |
| **脉脉** | 职场动态 | 职位浏览/薪资问答/职场八卦/人脉动态 |

### 行为数据结构

```python
@dataclass
class BehavioralData:
    platform: str                    # 平台来源
    user_id: str                     # 用户ID

    # 求职活跃度
    activity_level: ActivityLevel    # HIGH/MEDIUM/LOW
    last_active: datetime            # 最后活跃时间
    login_frequency: int             # 周登录次数
    avg_session_duration: float      # 平均会话时长(分钟)

    # 求职行为
    jobs_viewed_last_30d: int        # 近30天浏览职位数
    jobs_applied_last_30d: int       # 近30天投递职位数
    jobs_interested_last_30d: int    # 近30天收藏职位数
    search_queries: List[str]        # 搜索关键词

    # 简历行为
    resume_updated_at: datetime      # 简历更新时间
    resume_update_frequency: float    # 简历更新频率(次/月)
    profile_completeness: float      # 简历完整度(0-1)

    # 互动行为
    recruiter_response_rate: float   # HR消息回复率
    interview_attend_rate: float     # 面试赴约率
    offer_accept_rate: float         # Offer接受率

    # 意向信号
    intention_signals: List[str]     # ['actively_looking', 'passive_candidate', 'urgent']
    urgency_score: float             # 紧急程度(0-1)
    salary_expectation_trend: str     # 薪资预期变化(up/down/stable)
```

### 行为数据价值

```
┌─────────────────────────────────────────────────────────────────┐
│                   行为数据 → 求职阶段判断                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  活跃度HIGH + 回复率>80%      → 🟢 主动型人才、优先推荐           │
│  活跃度MEDIUM + 最近更新简历   → 🟡 观望中、可以接触             │
│  活跃度LOW + 3个月未更新      → 🔴 沉默候选人、需要激活          │
│                                                                 │
│  申请/浏览 > 10个/月          → 🟢 高意向、积极求职              │
│  申请/浏览 < 3个/月           → 🟡 被动型人才、需要包装           │
│                                                                 │
│  简历更新频率升高            → 🟡 可能在看新机会                  │
│  突然更新简历+完善资料       → 🔴 紧急求职、可能已离职           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 7. 隐私保护设计

### 数据安全措施

| 措施 | 说明 |
|------|------|
| **Token加密** | OAuth Token使用AES-256加密存储 |
| **最小权限** | 只申请必要的数据权限 |
| **数据脱敏** | 敏感信息（电话/邮箱）脱敏展示 |
| **本地处理** | 对话内容不传第三方，只提取特征 |
| **用户控制** | 用户可随时撤销授权、删除数据 |
| **审计日志** | 记录所有数据访问操作 |

### 用户删除权

```
┌─────────────────────────────────────────────────────────────────┐
│  🗑️ 删除我的数据                                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  [删除对话数据]  删除所有平台的历史聊天记录                       │
│  [删除行为数据]  删除所有平台的求职行为记录                       │
│  [删除简历数据]  删除所有平台的简历信息                          │
│  [全部删除]     删除TalentAI Pro中的所有数据                    │
│                                                                 │
│  ⚠️ 删除后无法恢复，确认删除？                                   │
```

## 8. TalentAI Skills 整合

### 数据流向

```
OAuth获取数据 → 清洗/去重/合并 → Skills分析 → 输出洞察
                                              │
┌─────────────────┬─────────────────┬─────────┴─────────┐
│ JD Intelligence │Candidate Intel │ Discovery Radar   │
│                 │                │                   │
│ • 解析JD中的    │ • 候选人活跃度  │ • 超预期发现      │
│   技能要求      │ • 求职意向      │ • 沉默候选人      │
│ • 匹配度计算    │ • 薪酬预期      │ • 流失风险        │
│ • 竞争分析      │ • 响应率预测    │ • 紧急求职信号    │
└─────────────────┴─────────────────┴─────────────────┘
```

### Skills使用对话/行为数据的场景

| Skills | 使用的数据 | 产出 |
|--------|-----------|------|
| **Candidate Intelligence** | 活跃度/回复率/简历更新频率 | 候选人意向评分 |
| **Matching Engine** | 申请记录/收藏记录/搜索词 | 优化匹配权重 |
| **Discovery Radar** | 对话信号/offer状态/响应时间 | 发现超预期候选人 |
| **Smart Outreach** | 历史沟通/响应率/活跃时间 | 优化触达时机和话术 |
| **Deal Tracker** | 面试记录/offer讨论/反馈 | 预测成交概率 |
