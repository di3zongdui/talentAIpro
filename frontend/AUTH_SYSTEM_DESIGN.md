## 三、登录系统设计

### 3.1 登录流程

```
┌─────────────────────────────────────────────────────────────────┐
│                        登录流程                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   用户访问 → login.html                                          │
│       │                                                        │
│       ├── 输入用户名/密码                                       │
│       │                                                        │
│       ├── 点击登录 → 后端验证                                    │
│       │        │                                               │
│       │        ├── 成功 → JWT Token → 跳转对应Dashboard          │
│       │        │                                               │
│       │        └── 失败 → 错误提示（剩余尝试次数）               │
│       │                                                        │
│       └── 忘记密码 → 邮箱重置链接                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 认证方式

| 方式 | 说明 | 适用场景 |
|------|------|---------|
| **用户名+密码** | 传统方式 | 所有用户 |
| **邮箱+密码** | 邮箱作为用户名 | 所有用户 |
| **手机号+验证码** | 免密登录 | 候选人 |
| **企业SSO** | 飞书/钉钉OAuth | 管理员/顾问 |

### 3.3 Demo模式

无需注册，直接选择角色进入体验：

```
┌─────────────────────────────────────────────────────────────────┐
│  🎯 快速体验 TalentAI Pro                                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  [👤 候选人]  [👔 猎头顾问]  [⚙️ 管理员]  [🔐 API开发者]        │
│                                                                 │
│  ⚠️ Demo模式数据为模拟数据，不可保存                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 四、数据模型

### 4.1 用户模型

```python
@dataclass
class User:
    id: str                           # UUID
    username: str                     # 用户名
    email: str                        # 邮箱
    phone: Optional[str]              # 手机号
    password_hash: str                # 密码哈希
    role: UserRole                    # 角色

    # 基础信息
    name: str                         # 姓名
    avatar: Optional[str]             # 头像URL
    status: UserStatus                # active/inactive/locked

    # 时间戳
    created_at: datetime
    updated_at: datetime
    last_login: Optional[datetime]

    # 权限
    permissions: List[str]            # 权限列表
    custom_dashboard: Optional[str]    # 自定义仪表盘
```

### 4.2 角色定义

```python
class UserRole(Enum):
    CANDIDATE = "candidate"           # 候选人
    CONSULTANT = "consultant"         # 猎头顾问
    ADMIN = "admin"                   # 管理员
    DEVELOPER = "developer"           # API开发者
    GUEST = "guest"                   # 演示访客
```
